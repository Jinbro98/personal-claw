# CE 파이프라인을 통해 얻은 학습

**카테고리:** best-practices
**날짜:** 2026-04-08
**프로젝트:** personal-claw (OpenClaw 개인화 플러그인)

## CE 파이프라인 실행 기록

```
CE:brainstorm → CE:plan → CE:work → CE:review → CE:compound
```

## 학습 내용

### 1. 멀티 에이전트 병렬 실행의 효과

- Unit 2+3+4 병렬: 3개 에이전트가 동시에 작업 → 시간 ~60% 절감
- Unit 5+6과 Unit 7+8 병렬: 추가 시간 절감
- **교훈:** 독립적인 작업 단위는 반드시 병렬로. 의존성 그래프를 먼저 그리면 파악 가능.

### 2. CE:review의 가치

- P0 버그를 발견하지 못했다면 전체 학습 파이프라인이 no-op인 채로 배포될 뻔
- 테스트가 통과해도 통합 경로의 버그는 잡지 못함
- **교훈:** 단위 테스트 + 리뷰어 페르소나 = 필수 조합

### 3. 불변 패턴 + TypeScript의 함정

- TypeScript는 반환값 무시를 컴파일 타임에 잡지 않음
- 불변 함수를 mutable 패턴으로 사용하는 실수 빈번
- **교훈:** 네이밍(with/clone) + 린트 + 통합 테스트로 삼중 방어

### 4. 한국어 정규식의 어려움

- "잘 못"의 두 가지 의미 (긍정/부정)
- 교착어 특성상 단어 경계가 모호
- **교훈:** 한국어 패턴은 반드시 다양한 문장으로 교차 검증

### 5. CE:compound의 복리 효과

- 첫 해결: 리서치 30분 + 디버깅 30분
- 문서화: 5분
- 다음 발생: 빠른 조회 2분
- **교훈:** 시간이 지날수록 해결 속도가 빨라진다

## 파이프라인 개선 제안

1. CE:work에서 통합 테스트를 필수 단위로 포함
2. CE:review에서 P0 발견 시 자동 수정 제안
3. CE:compound에서 자동 카테고리 분류 (현재 수동)
