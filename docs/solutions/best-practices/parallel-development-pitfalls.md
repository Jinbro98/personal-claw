# 병렬 개발의 함정과 해결법

**카테고리:** best-practices
**날짜:** 2026-04-08
**프로젝트:** personal-claw v2

---

## 문제

v2에서 Unit 1+3+4를 병렬로 개발하면서 발생한 대표적 문제:

1. **import ≠ 연결**: Unit 5+6에서 함수를 추가하고 import를 했지만, Unit 2에서 파이프라인 연결 시 아직 완료되지 않아 연결 누락 (P0)
2. **훅 타입 불일치**: A/B 제안을 `onMessage`에서 전달하려 했지만, 반환 타입이 `void` (P1)
3. **패턴 false positive**: 주제 감지의 `/글/i`가 한국어 거의 모든 문장에서 매칭 (P2)

## 패턴

### 패턴 1: 인터페이스 먼저 확정

병렬 개발 전에 각 모듈 간 인터페이스(함수 시그니처, 타입)를 확정해야 한다.

```
❌ 병렬로 바로 코딩 시작
✅ 1. 인터페이스 정의 → 2. 스텁 코드 공유 → 3. 병렬 구현
```

### 패턴 2: 훅/API 타입 먼저 확인

플러그인 시스템의 hook 타입을 먼저 확인하고 전달 방식을 결정.

```
❌ "어떻게든 전달하겠지" → console.log
✅ hook 타입 확인 → 반환 타입이 void이면 다른 경로(시스템 프롬프트) 사용
```

### 패턴 3: 한국어 정규식은 구체적으로

```
❌ /글/i → "대단합니다", "그렇습니다" 모두 매칭
✅ /글쓰기|글\s*작성|글\s*써/i → 의도한 패턴만 매칭
```

### 패턴 4: import 미사용 감지

빌드 시 import만 되고 미사용인 코드를 자동 감지하는 린트 규칙 필요.

## 적용 시점

- 3개 이상의 독립 모듈을 병렬로 개발할 때
- 플러그인/훅 기반 아키텍처에서 개발할 때
- 한국어 정규식 패턴을 작성할 때

## 관련

- P0: buildTopicAwarePrompt 미연결
- P1: A/B 제안 console.log 출력
- P2: /글/i false positive
