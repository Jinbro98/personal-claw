# 정규식 false positive — 부정 표현이 긍정으로 감지

**카테고리:** runtime-errors
**날짜:** 2026-04-08
**심각도:** P1 (높음)
**영향:** 감정 분석 오류, 잘못된 학습

---

## 문제

`signals.ts`의 `explicit_positive` 패턴에 `/잘\s*못/i`가 포함되어 있었다.

한국어에서 "잘 못"은 두 가지 의미:
1. "잘못하다" = 제대로 못하다 (부정) ❌
2. "잘 ~ 못하다" = "~를 잘 할 수 없다" (부정) ❌

긍정 패턴에 포함되면 "잘 못해" 같은 부정 메시지가 `reward: +1.0`으로 처리됨.

```typescript
// ❌ 잘못된 코드
patterns: [
  /잘\s*했/i,    // "잘 했어" = 긍정 ✅
  /잘\s*못/i,    // "잘 못해" = 부정이지만 긍정으로 감지 ❌
]
```

## 증상

- "나 이거 잘 못하겠어" → explicit_positive 감지 (reward +1.0)
- 사용자가 불만을 표현했는데 시스템은 만족으로 해석
- 잘못된 학습 → 프로필 왜곡

## 솔루션

긍정 패턴에서 `/잘\s*못/i` 제거.

```typescript
// ✅ 올바른 코드
patterns: [
  /잘\s*했/i,
  // /잘\s*못/i intentionally excluded (means "can't do well" = negative)
]
```

## 이유

한국어는 교착어라 동일 단어가 문맥에 따라 정반대 의미를 가질 수 있다. 정규식으로 감정을 분석할 때 특히 주의해야 한다.

## 예방

1. 한국어 패턴은 반드시 **다양한 문장**으로 테스트
2. 긍정/부정 패턴의 **교차 검증** (같은 패턴이 양쪽에 없는지)
3. 감정 분석 테스트 케이스에 엣지 케이스 포함
4. 정규식만으로 부족하면 키워드 + 문맥 분석 조합 사용
